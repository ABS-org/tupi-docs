Formatação

O formato de código escolhido deve garantir que o código seja: fácil de ler; fácil de comentar claramente; minimize a chance de introduzir erros acidentalmente; e resulte em úteis visualizações de diferença. Para isso, vale seguir as normas abaixo:

  • Use um seletor/tag por linha.
  • Um único espaço antes da abertura das chaves em um conjunto de regras.
  • Uma única declaração por linha.
  • Um nível de indentação para cada declaração e para cada bloco de tags.
  • Sempre inclua um ponto-e-vírgula no fim da última declaração em um bloco declarativo.
  • Separe cada conjunto de regras por uma linha em branco.
  • Use cores em valor hexadecial em vez de RGB.
  • Não especifique a unidade quando o valor for zero.
  • Esforce-se para definir os valores com apenas uma propriedade em propriedades específicas, ex.: margin, padding.

Veja alguns exemplos de boas e más práticas:

/* Ruim */
.seletor-1, .seletor-2, .seletor-3 {
  color: #333;
}

/* Bom */
.seletor-1,
.seletor-2,
.seletor-3 {
  color: #333;
}

/* Ruim */
.seletor {
  color: #333; margin: 0;
}

/* Bom */
.seletor {
  color: #333;
  margin: 0;
}
        
<!-- Ruim -->
<div class="box"><p class="paragraph"></p></div>

<!-- Bom -->
<div class="box">
  <p class="paragraph">

  </p>
</div>

<!-- Ruim -->
<div class="box">
<p class="paragraph">
</p>
</div>

<!-- Bom -->
<div class="box">
  <p class="paragraph">
  
  </p>
</div>
