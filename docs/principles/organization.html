Organização dos Arquivos

Organização de código é uma importante parte de qualquer base de código CSS, e crucial para grandes bases de código. Separar logicamente distintas partes do código. Abstrair partes comuns de código em variáveis para cor, tipografia, etc.

Iremos nos basear nos princípios abaixo para manter a uniformidade na organização:

A base irá conter todo elemento básico que será distribuído nas demais partes da aplicação. Ex.: mixins, grid, tipografia, padrão de botões, ícones etc.

A parte de estilos contém especificamente aspectos relacionados ao o visual. Nesse contexto, podemos ter um slider ou accordion, com seus mecanismos, ações, javascript etc, mas com um estilo específico. Assim, o slider da Home pode ter um estilo diferente de um outro slider na seção de usuários ou comunidades, por exemplo.

Em componentes está o CSS referente aos componentes que usam javascript. Ex.: slider, tabs etc.

Na raiz deve conter apenas o arquivo de variáveis e o arquivo para build.

Segue o esquema ilustrado abaixo:

  • base/
    • mixins.scss
    • grid.scss
    • icons/
      • icons-outline/
      • icons-other/
    • ...
  • styles/
    • slider-home.scss
    • accordion-users.scss
    • ...
  • components/
    • slider.scss
    • accordion.scss
    • tabs.scss
    • ...
  • _variables.scss
  • build.scss
